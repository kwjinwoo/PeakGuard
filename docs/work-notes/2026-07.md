# Work Notes — 2026-07

## 2026-07-02 — Instructions described obsolete JSON persistence

- Context: Hierarchical `AGENTS.md` files and the initial LLM Wiki were derived from `.github/copilot-instructions.md`.
- Difficulty: The old instructions described `peak_prices.json` and JSON serialization, while the current implementation, tests, and README use `peak_prices.csv`.
- Verification: Compared `src/peakguard/storage.py`, `src/peakguard/main.py`, `tests/test_storage.py`, and `README.md`.
- Conclusion: CSV in a GitHub Gist is the current source of truth; instructions must not reintroduce the obsolete JSON design.
- Related: [ADR-0001](../decisions/0001-csv-gist-persistence.md), [Data contracts](../concepts/data-contracts.md).

## 2026-07-02 — Z-score documentation overstated runtime behavior

- Context: The README lists a Z-score alert alongside active report signals.
- Difficulty: Configuration and domain calculation exist, but orchestration and `TickerSummary` do not use the Z-score threshold.
- Verification: Traced `zscore_threshold` and `calculate_price_zscore` references through `src/peakguard` and tests.
- Conclusion: Document Z-score as partially implemented until orchestration, report formatting, and tests are completed or the unused feature is removed.
- Related: [Alert catalog](../concepts/alerts/README.md), [Next](../roadmap/next.md).
